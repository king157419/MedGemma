# MoonBit 2026 软件合成大赛参赛计划

## 一、项目概述

**项目名称**：MedReport Bridge（医疗报告解读助手）

**核心功能**：家属输入患者的医疗报告单（文本或影像），系统输出通俗易懂的健康状况解读，降低医患沟通成本。

---

## 二、技术架构

### 2.1 系统组成
```
┌─────────────────────────────────────────────────────────┐
│                    MoonBit 核心应用                      │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐  │
│  │  报告解析器  │  │  AI服务网关  │  │  结果格式化器   │  │
│  │  (Parser)   │  │  (Gateway)  │  │  (Formatter)   │  │
│  └─────────────┘  └─────────────┘  └─────────────────┘  │
├─────────────────────────────────────────────────────────┤
│              MoonBit 异步HTTP服务 + 结构化并发            │
└─────────────────────────────────────────────────────────┘
                          ↓ HTTP API
┌─────────────────────────────────────────────────────────┐
│              MedGemma 模型服务 (Python/FastAPI)          │
│              部署在云服务器 或 本地GPU                    │
└─────────────────────────────────────────────────────────┘
```

### 2.2 技术选型
| 组件 | 技术方案 | 说明 |
|------|----------|------|
| 核心应用 | MoonBit | 异步HTTP服务，报告解析，结果格式化 |
| AI模型 | MedGemma 4B/27B | 开源医疗模型，支持文本和影像 |
| 模型部署 | HuggingFace + vLLM | 本地/云端部署 |
| 前端 | 简洁Web界面 | 用户交互 |

## 五、项目申报书要点

### 5.1 应用场景
- 患者家属收到医疗报告单后难以理解专业术语
- 医生时间有限，无法详细解释每项指标
- 需要快速了解患者健康状况

### 5.2 核心功能
- 多格式报告输入（文本/图片/PDF）
- 智能解读生成（使用MedGemma）
- 通俗化输出（面向非专业人士）

### 5.3 风险与应对
| 风险 | 应对方案 |
|------|----------|
| MedGemma非临床级别 | 明确标注"仅供参考，不作为诊断依据" |
| 模型部署成本 | 使用云服务按需付费，或本地GPU |
| MoonBit生态不完善 | 参考官方示例，必要时贡献代码 |
